# ADR-006: Why no-std with WASM target

## Context

Soroban contracts run in a WASM environment without access to the standard Rust library.

The dependency model and runtime constraints differ significantly from conventional native Rust applications.

## Decision

Target `no-std` with the WASM contract target to ensure compatibility with Soroban execution semantics.

## Consequences

- Enables deployment on Stellar's Soroban runtime.
- Requires use of `soroban-sdk` abstractions and avoidance of standard library features.
- Simplifies contract portability and reduces runtime dependencies.
