# ADR-005: Actor verification caching strategy in shared module

## Context

Many contracts require repeated verification of actor roles, permissions, and contract authorship.

Repeated checks can increase gas costs and reduce performance if each call re-evaluates the same actor context.

## Decision

Cache verified actor assertions in a shared module to avoid redundant verification work during the same transaction or request flow.

## Consequences

- Improves performance for workflows with repeated actor checks.
- Preserves strong authorization guarantees by caching only within a transaction boundary.
- Requires careful invalidation semantics and auditability of cached decisions.
