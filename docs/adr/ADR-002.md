# ADR-002: Storage TTL retention class design (Critical/Operational/Ephemeral)

## Context

Healthcare contracts must retain data for varying durations depending on legal requirements and storage cost sensitivity.

A unified TTL model is needed to avoid uncontrolled state growth while preserving critical records.

## Decision

Implement three retention classes:
- Critical: long-lived records with the largest ledger bump and highest threshold.
- Operational: medium-term state for business workflows.
- Ephemeral: short-lived records for temporary or cache-like data.

## Consequences

- Simplifies storage rent policies by tiering state retention.
- Allows contracts to choose the appropriate tradeoff between persistence and cleanup pressure.
- Requires comprehensive testing to verify read/write bump behavior and bulk expiration semantics.
