# ADR-001: Why Soroban/Stellar vs other blockchain platforms

## Context

This repository targets a decentralized healthcare platform that must support high-throughput smart contracts, deterministic execution, and integration with existing financial rails.

Existing options such as Ethereum, Hyperledger, or private blockchains impose higher gas costs, slower finality, or more complex permissioning than needed for this use case.

## Decision

Use Soroban on Stellar as the primary smart contract platform.

## Consequences

- Enables WASM-based Rust contracts with deterministic execution.
- Leverages Stellar's financial primitives and low-cost transaction model.
- Provides a simpler deployment path for healthcare workflows than EVM-based platforms.
- Requires adoption of Soroban tooling and Stellar network conventions.
